<h2 style="color:#0786b4;">Q1 — O que descreve melhor Pensamento Computacional (PC)?</h2>

<b>Timer:</b> 30s · <b>Points:</b> On

<b>Alternativas:</b>
1. Um conjunto de habilidades para <b>resolver problemas, projetar sistemas e compreender processos</b> usando conceitos fundamentais da Computação. ✅ <b>Correta</b>  
2. Sinônimo de programar em qualquer linguagem.  
3. Técnica para automatizar tarefas sem modelagem.  
4. Domínio de sintaxe para escrever código mais rápido.  

<b>Feedback:</b>  
Definição clássica de PC: abordagem para resolver problemas, projetar sistemas e compreender processos com base em conceitos da Computação.

<b>Referência:</b> Wing (2006). [cs.cmu.edu]

<hr style="border:0; border-top:1px solid #0786b4; opacity:0.25;"/>

<h2 style="color:#0786b4;">Q2 — No framework de Brennan &amp; Resnick, qual combinação está correta?</h2>

<b>Timer:</b> 25s · <b>Points:</b> On

<b>Alternativas:</b>
1. <b>Conceitos:</b> sequências, laços, condicionais · <b>Práticas:</b> iterar, depurar · <b>Perspectivas:</b> expressar, conectar, perceber-se como criador. ✅ <b>Correta</b>  
2. <b>Conceitos:</b> estética, narrativa · <b>Práticas:</b> vender jogo · <b>Perspectivas:</b> monetizar.  
3. <b>Conceitos:</b> personagens, arte · <b>Práticas:</b> publicar · <b>Perspectivas:</b> competir.  
4. <b>Conceitos:</b> shaders, engine · <b>Práticas:</b> otimizar · <b>Perspectivas:</b> terceirizar.  

<b>Feedback:</b>  
As três dimensões do framework são <b>conceitos</b>, <b>práticas</b> e <b>perspectivas</b>; amplamente usadas em estudos de criação de mídia interativa e transferíveis para atividades de jogos.

<b>Referência:</b> Brennan &amp; Resnick (2012). [scratched...arvard.edu]

<hr style="border:0; border-top:1px solid #0786b4; opacity:0.25;"/>

<h2 style="color:#0786b4;">Q3 — Na perspectiva do designer no MDA, qual é o encadeamento correto?</h2>

<b>Timer:</b> 30s · <b>Points:</b> On

<b>Alternativas:</b>
1. Estéticas → Dinâmicas → Mecânicas.  
2. Dinâmicas → Mecânicas → Estéticas.  
3. <b>Mecânicas</b> (controle direto do designer) → <b>Dinâmicas</b> (comportamentos em runtime) → <b>Estéticas</b> (respostas do jogador). ✅ <b>Correta</b>  
4. O designer controla simultaneamente estéticas e mecânicas.  

<b>Feedback:</b>  
No MDA, o designer atua principalmente nas <b>mecânicas</b>; delas emergem <b>dinâmicas</b> e, por fim, as <b>estéticas</b> percebidas pelo jogador.

<b>Referência:</b> Hunicke, LeBlanc &amp; Zubek (2004). [cs.northwestern.edu]

<h2 style="color:#0786b4;">Q4 — Sobre FSM (Máquinas de Estados Finitos) em jogos, qual afirmação é verdadeira?</h2>

<b>Timer:</b> 25s · <b>Points:</b> On

<b>Alternativas:</b>
1. Uma FSM tem um conjunto <b>finito</b> de estados, <b>apenas um ativo por vez</b>, e transições condicionadas por eventos/regras. ✅ <b>Correta</b>  
2. Uma FSM permite infinitos estados ativos simultaneamente, sem transições.  
3. FSM é apenas visual; não pode ser implementada em código.  
4. FSM não é útil para IA ou UI.  

<b>Feedback:</b>  
FSM organiza comportamento em estados mutuamente exclusivos com <b>transições</b> claras — muito útil para IA, UI e controle de personagem.

<b>Referência:</b> <i>Game Programming Patterns</i> — capítulo <b>State</b>. [gameprogra...tterns.com]

<h2 style="color:#0786b4;">Q5 — Qual vantagem do Portugol Studio é mais relevante para prototipagem inicial em cursos de jogos?</h2>

<b>Timer:</b> 20s · <b>Points:</b> On

<b>Alternativas:</b>
1. Ambiente em português com <b>exemplos</b>, <b>depurador interativo</b> e <b>bibliotecas para jogos</b> (loops, eventos, desenho). ✅ <b>Correta</b>  
2. É restrito a aplicações web e não suporta jogos.  
3. Foco exclusivo em sintaxe avançada e sem materiais didáticos.  
4. Não permite depuração passo a passo.  

<b>Feedback:</b>  
O site/documentação do Portugol Studio destacam recursos didáticos, depuração e suporte a bibliotecas para criação de jogos — ideal para iniciantes.

<b>Referência:</b> Site oficial UNIVALI‑LITE. [Portugol Studio]

<h2 style="color:#0786b4;">Passo a passo para publicar no Kahoot!</h2>

1. Acesse <b>kahoot.com</b> → <b>Log in</b> → <b>Create</b> → <b>Quiz</b>.  
2. Copie cada pergunta acima, <b>selecione a alternativa correta</b> e ajuste <b>Timer</b> e <b>Points</b>.  
3. Em <b>Game options</b> antes de iniciar: ative <b>Shuffle answers</b>, <b>Friendly nickname</b> e, se quiser, <b>Two-step join</b>.  
4. <b>Start</b> em modo <b>Classic</b> (ou <b>Teams</b>) → compartilhe o <b>PIN</b> na sala/Teams.  
5. Ao final, baixe o <b>relatório</b> para analisar acertos por questão (ótimo para mapear conceitos que precisam de reforço).
